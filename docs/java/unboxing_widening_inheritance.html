Java method invocation: boxing/unboxing versus widening versus inheritance

If we call a method and Java doesn’t find the exact method signature then Java will check if it can call another method with a similar signature using inheritance. In the case of signature containing numbers, Java will use boxing/unboxing and widening.

Let’s have a quick look at these concept one by one.

  • Inheritance.  In the example below, which Test method is going to be called?

public static void main(String args[])
{
BoxingWideningTest t= new BoxingWideningTest();
Short c=1;
t.Test(c);
}
public void Test(Integer c) {System.out.println(“Class Integer called”);}
public void Test(Object c) {System.out.println(“Class Object called”);}
}

We are calling a method with a Short class parameter. As a method with the “Short” signature is unavailable, Java will call the method with the Object class as call Object is a parent of class Short. So “Class Object called” will be printed.

  • Boxing/unboxing: For example:

public static void main(String args[])
{
BoxingWideningTest t= new BoxingWideningTest();
Short c=1;
t.Test(c);
}
public void Test(short c) {System.out.println(“Primitive short called”);}
public void Test(Integer c) {System.out.println(“Class Integer called”);}

In the example above, Java will use unboxing and call the method with the primitive short. So “Primitive short called” will be printed.

  • Widening. For example:

public static void main(String args[])
{
BoxingWideningTest t= new BoxingWideningTest();
short c=1;
t.Test(c);
}
public void Test(int c) {System.out.println(“Primitive int called”);}
public void Test(long c) {System.out.println(“Primitive Long called”);}

Please note the method contains now a “short” primitive parameter and not a “Short” class anymore.

This applies only for primitive. Java will widen the type in order to find a matching signature. In the case of the short, Java will widen to integer and find a match. “Primitive int called” will be printed.

This is all good and we easily find documentation about these 3 concepts.

But what if we mix all these 3 concepts together in the same code?

Let’s look at the following code:

public static void main(String args[])
{
BoxingWideningTest t= new BoxingWideningTest();
Short c=1;
t.Test(c);
}
public void Test(short c) {System.out.println(“Primitive short called”);}
public void Test(int c) {System.out.println(“Primitive int called”);}
public void Test(long c) {System.out.println(“Primitive Long called”);}
//public void Test(Short c) {System.out.println(“Class Short called”);}
public void Test(Integer c) {System.out.println(“Class Integer called”);}
public void Test(Object c) {System.out.println(“Class Object called”);}

Now, which method is going to be called? Is inheritance going to have priority over boxing? Is Java capable of doing unboxing and widening at the same time? Which method can be called and in which order? It turns out the text printed will be “Class Object called”.

The order depends whether the signature contains a class or a primitive.
